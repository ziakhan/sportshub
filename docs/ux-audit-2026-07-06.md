## Method

Owner directive (2026-07-06): stop ad-hoc gap-chasing; audit every menu and
every page systematically, then fix in prioritized batches. Lean hybrid
execution: route inventory + expectations baseline built inline; six parallel
agents walk the app as real personas (login recipe, GET-only); the main
session verifies key findings against code and synthesizes one ledger.

- Scope: all personas, all routes (~80 pages), menus at code level.
- Judged against: the expectations baseline below (spec docs + IA plan
  research: SportsEngine / TeamSnap / GameChanger / ESPN patterns).
- Out of scope by design: visual design opinions, performance, security,
  new features.

## Expectations baseline (what "no gaps" means per persona)

| Persona | Must be able to (jobs-to-be-done) |
|---|---|
| Anonymous visitor | Browse leagues/clubs/teams/players freely; scores, standings, leaders, news ≤2 clicks from home; every entity links to related entities; graceful 404s |
| Parent | See each kid → team → schedule → stats chain from anywhere kid appears; offer + payment + registration history; clear labels on personalized surfaces; shallow account admin without entering operator world |
| Player (13+) | "My stats / my team / my schedule" reachable from home; own public page discoverable |
| Club owner | Full club ops from one workspace; every managed entity links to its public page ("view as public"); paper trail: tryout → offer → roster → payment |
| League owner | Season ops (divisions/schedule/standings/fees); jump between manage world and public hub in one click; same numbers in both worlds |
| Referee | Assignments list, profile/PIN, scoresheets of assigned games |
| Platform admin | Console reachable; every admin surface in the menu |

## Findings ledger

### Verified critical (main-session confirmed, corroborated by multiple auditors)

1. **Every public club page 500s** — `/club/[slug]` queries `HouseLeague.ageGroup`
   but the model field is `ageGroups` (plural). Confirmed live (500 on
   /club/416-united) + root cause in page code. Every club link on the site
   (homepage featured clubs, team pages, news tags, directory) dead-ends.
   Corroborated by 5/6 auditors. **Recommend hotfix immediately.** (S)
2. **Sidebar "My Teams" → `/teams` = 404** — nav-config Player and Staff
   sections link a route that does not exist. (S)
3. **Referee gets NO navigation chrome** — confirmed: zero sidebar/topnav/
   sign-out markers on any platform page as referee@sportshub.test. Likely
   nav builds no sections for referee-only users and the layout renders
   nothing. (M)
4. **`/score` (scoring hub) is orphaned** — grep confirms zero menu/page
   references; the entry point to live scoring is type-the-URL-only. (S)
5. **Marketplace shows "No tryouts" while 38-39 exist** — filter mismatch
   between marketplace query and homepage/API. (M)
6. **Live game pages are link dead-ends** — no team/player/league anchors on
   /live/[gameId]; combined with /scores linking only to /live, a fan can
   never reach a team page from the scores flow. (M)
7. **Season-manage workspace (9 tabs) has zero outbound links** — games/
   teams/standings rows are plain text; no path to live view, scoresheet,
   console, or public pages. (M)

### Adjudicated: works-as-designed or demo-data (not code bugs)

- **"Minors' full names shown publicly"** (multiple auditors): the flagged
  players have `mediaConsent=GRANTED` — full names ARE the consented
  behavior. Real issue: the showcase seeder granted consent to ~1/3 of
  players including both demo kids, making the privacy default invisible.
  → seeder change: demo kids UNSET. Roster "mixed" naming = consent working.
- **"Duplicate recaps of the same game"**: slugs contain different game-id
  fragments — these are distinct seeded games with identical matchups from
  repeated demo-script runs, not a recap-engine dupe. → demo-data noise.
- **Bracketed test-world names on /leagues and news** ([wysmp16] etc.):
  integration-test worlds re-created after the showcase scrub. → recurring
  demo hygiene; consider excluding test worlds from public queries.
