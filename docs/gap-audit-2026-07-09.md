# 🕳️ Platform gap audit — "what else are we missing?" (2026-07-09)

**Owner ask:** ad-hoc development keeps leaving gaps (editability audit proved it). Sweep the
whole site + codebase for the obvious missing things. **Method:** 5 parallel read-only agents,
one lens each — navigation dead-ends · email/notification coverage · robustness · authz drift ·
dead code & stubs. ~130 API handlers and ~200 internal links traced. Companion docs:
[[editability-audit]] (lifecycle/actions) · [[season-continuity-plan]] (rollover + comms/consent).

## Executive summary — the three systemic findings

1. **The communication layer is club-biased and family-silent.** Clubs get bells for everything;
   the *paying family* gets nothing at the moments that matter most: no registration
   confirmations, **no receipt for the offer-acceptance payment**, **refunds/waives completely
   silent**, game cancellations bell-only (the "phone-tree replacement" doesn't email), final
   scores never reach families. And there is **zero unsubscribe/CASL infrastructure** while we
   already blast practice/event emails to every member. → Fix as one program: the consent +
   messaging foundation in [[season-continuity-plan]] §5 now doubles as transactional repair.
2. **Global venues were an authz blind spot.** Venue PATCH was fixed this session, but
   courts/hours mutation endpoints (session-only, global data) were missed — plus a repeating
   IDOR pattern where child-delete endpoints don't verify the child belongs to the parent.
   → **All fixed 2026-07-09** (see §2).
3. **Modeled-but-never-enforced features masquerade as built.** `TenantFeatures` (the entire
   plan/entitlement system) is created per-tenant and read by nothing; league payment-config
   APIs have no UI; `PaymentStatus.DISPUTED` has no chargeback handling; `hasFamilyPass()` is a
   hardcoded `true`. The monetization layer largely *looks* done in schema and isn't.

### P2 — hygiene / consistency
- Reviews: any user can review any club/person with no relationship check (product decision) + the
  empty `api/reviews/[id]/` scaffold (edit/delete/moderation = launch blocker H4).
- `GET /api/clubs/[id]` returns org contact fields to any signed-in user (mostly-public info; tighten or select-public).
- Status→badge-tone maps hand-rolled ~10×; centralize in the kit (design-system wave 2 item).
- Dead surface cleanup: orphaned `/api/dev/create-test-users`, dead team-level offer-template routes,
  empty `(platform)/marketplace/` dir, style-guide page ship-gating, dev util pages.
- New dead enums beyond editability-audit §3: `PaymentStatus.DISPUTED` (no chargeback path),
  `PostKind.ARTICLE/PHOTO_SET` (+VIDEO never written), `MediaAssetType.VIDEO_NATIVE`,
  `CampSignupStatus.WAITLISTED/CONFIRMED`. (§3's StaffInvitation line now stale — CANCELLED/EXPIRED live.)
- Club-create success CTA hardcodes `http://{subdomain}/dashboard`; add in-app `/clubs/[id]` CTA.
- `teams/[id]/finalize` has no already-finalized guard (idempotent-ish; note only).
- DOB round-trip via `toISOString().split("T")` (safe only while stored as UTC-midnight).
- Impersonation attribution: staff-invite `invitedById` stamps the real admin, not the impersonated owner.
- No `Tenant.isTestData` flag (demo worlds identified heuristically).

## 2. Fixed same-day (2026-07-09, local)
- Venue **courts POST/PATCH/DELETE + hours PUT** gated to operator roles (ClubOwner/ClubManager/
  LeagueOwner/LeagueManager) or PlatformAdmin — venue-editor UI (league manage) keeps working;
  random members can no longer vandalize global venues.
- **6 IDOR scopings**: season divisions/venues/sessions DELETE + tournament divisions/venues DELETE
  + tournament teams PATCH now verify the child row belongs to `params.id`.
- **Onboarding checklist 404s**: season/division steps now point at `/manage/leagues/[id]` and
  `.../seasons/[seasonId]/manage`.
- **Sign-in round-trip**: tryout page now passes `callbackUrl`.
